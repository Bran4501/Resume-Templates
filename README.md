## 🔧 What to edit first

Start with the basic sections that matter most.

### Personal details

Add:

- your name
- your email
- your phone number
- your city and state
- your LinkedIn profile
- your GitHub profile if you have one

### Summary

Write a short summary that matches the role you want. Keep it plain and direct. For example:

- Software engineer with experience in Python, web apps, and data tools
- AI/ML student with hands-on work in model training and deployment
- Fullstack developer with front-end and back-end project experience

### Skills

List skills in groups:

- Languages: Python, Java, C++, JavaScript
- Tools: Git, Docker, Linux
- Web: React, Node.js, HTML, CSS
- Data: NumPy, pandas, scikit-learn
- Cloud: AWS, Azure, GCP

### Work history

For each job, add:

- job title
- company name
- dates
- 2–4 bullet points
- clear results where possible

Use strong verbs like:

- built
- improved
- reduced
- shipped
- designed
- led

### Projects

This section matters a lot for students and early-career applicants. Add projects that show real work.

Each project should include:

- project name
- short description
- tools used
- what you built
- what result it gave

## 🚀 How to use the templates well

A good resume is not just about design. It is about clarity.

### Keep the layout simple

Do not crowd the page. Leave space around sections so the resume is easy to scan.

### Match the job post

Use the same words from the job description when they fit your experience. If the role asks for Python, include Python. If it asks for React, include React.

### Keep it one page if you can

For most early-career roles, one page is enough. If you have many years of work history, two pages can work too.

### Use short bullet points

Each bullet should say one thing. Keep it focused on action and result.

Good example:

- Built a resume parser that cut manual review time by 35%

Weak example:

- Worked on many things for the team and helped improve the system in several ways

## 🎯 Tips for ATS-friendly resumes

ATS systems read text and search for matches. Keep the resume easy to parse.

Use these tips:

- use standard section names like Experience, Education, Skills, Projects
- avoid heavy graphics
- avoid tables if they break the layout
- use plain job titles
- keep dates in a clear format
- use common skill names

Example date format:

- Jan 2023 – May 2024

Example section names:

- Experience
- Projects
- Education
- Skills

## 🛠️ Troubleshooting on Windows

If the PDF does not build:

- reopen the editor as normal user or admin
- check that the file path has no odd characters
- install missing LaTeX packages when prompted
- try a second build after the first run
- make sure the correct `.tex` file is open

If text cuts off:

- shorten long bullet points
- reduce the amount of content in one section
- remove old projects that no longer help
- move low-value items off the page

If fonts look wrong:

- rebuild the PDF
- confirm the font package is installed
- use the template defaults first before changing styles
